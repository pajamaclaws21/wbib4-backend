How to

Primarily, I'd strongly recommend reading the docs from browser.engineering. That is where I got most of the design inspiration from. There aren't many things you need to change; this is pretty drag-and-drop; but two things stick out.
  • allowedOrigins: the way the app is set up now, if an origin that is not on this list makes an api call, they recieve an error 401. please change this to your desired origins.
  • user-agent: self-explanatory. Please change the userAgent variable to something different than "pajamaclaws-net"!!!
  • OVERALL: I do believe I learned a decent amount making this project. However, younger me would've just forked the app and deployed it without making the changes or really understanding what the code did. so yeah if you identify with that i'd recommend reading the code. I really am still learning abt this too
  • MASSIVE CHANGES: if you are making massive changes, please also change the uaName variable to your project name, and the uaVersion variable to your project version. I am trying to make it as easy as possible for sites&servers to distinguish between different forks.
  • On that note: if you make a cool fork and you think the features should be in the main please make a pull request!! I would be delighted!
